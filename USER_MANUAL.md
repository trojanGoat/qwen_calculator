# User Manual

## Workflow

1. **Initialization**: Initialize the git repository and set the remote to GitHub.
2. **Coding Phase**: Interact with OpenCode to build the application.
3. **Testing**: Run `./launch_test.sh` to ensure everything works correctly.
4. **Committing**: Let the agent push and pull code directly, or review the diffs and commit yourself.
5. **Documentation**: Update the `CHANGELOG.md` as new features are added.

## OpenCode Llama Cluster

The system relies on a locally hosted Llama cluster (`192.168.8.1:8080`). Ensure the cluster is up and running before initiating OpenCode tasks.
